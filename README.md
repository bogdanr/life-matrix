# Life Matrix ESPHome Component

A time-passage visualization component for ESPHome and HUB75 LED matrix displays. Renders year, month, day, and hour progress as pixel art — with biographical lifespan phases and a Game of Life panel.

## Screens

1. **Year View** — Calendar heatmap of all 365 days with event markers and a breathing "today" indicator
2. **Month View** — 4×8 day grid with activity fill, event borders, and current-time progress in today's cell
3. **Day View** — 24h visualization split into Sleep / Work / Life segments with rainbow coloring
4. **Hour View** — Spiral fill with multiple color schemes (seasonal, gradient, rainbow, single)
5. **Lifespan View** — Biographical visualization with life phases (school, career, retirement), relationship markers, and cosmos starfield beyond life expectancy
6. **Game of Life** — Conway's cellular automaton with age-based coloring and smart auto-reset
7. **Pomodoro Timer** — Work/break timer with spiral visualization, exercise snack reminders

## Hardware

- **ESP32-S3** (recommended for performance)
- **HUB75 RGB LED Matrix** — tested with 2× 64×32 panels chained horizontally, rotated 90° → 32×120 effective
- **Tested on**: Adafruit Matrix Portal S3
- **Optional**: Rotary encoders, buttons for physical UI

## Installation

### From GitHub

```yaml
external_components:
  - source: github://bogdanr/life-matrix
    components: [ life_matrix ]
```

### Local Development

Copy the component files (`__init__.py`, `life_matrix.cpp`, `life_matrix.h`) to `esphome/components/life_matrix/` in your ESPHome config directory.

## Quick Start

See [`example-minimal.yaml`](example-minimal.yaml) for a basic setup or [`example-full.yaml`](example-full.yaml) for a complete configuration with HA integration and physical UI. The essentials:

```yaml
life_matrix:
  id: life_matrix_component
  display: matrix_display    # Wire brightness callback
  time_id: sntp_time
  font_small: font_sm
  grid_width: 32
  grid_height: 120
  screen_cycle_time: 5s
```

Then call `render()` from your display lambda:

```yaml
display:
  - platform: hub75
    id: matrix_display
    # ... your panel config ...
    lambda: |-
      auto time = id(sntp_time).now();
      if (time.is_valid()) {
        id(life_matrix_component)->render(it, time);
      }
```

## Home Assistant Entities (Auto-Generated)

The component auto-generates 47 entities for full remote control:

| Type | Count | Entities |
|------|-------|----------|
| **Switches** | 11 | Screen toggles (8), show_future, complex_patterns, exercise_snacks |
| **Selects** | 10 | style, gradient_type, fill_direction, marker_style, marker_color, day_fill, year_event_style, conway_speed, pomodoro_preset, text_area_position |
| **Numbers** | 12 | brightness, night_mode, cycle_time, bed_time, work_start, work_end, pomodoro_rounds, moved_out, school_years, retirement, life_expectancy, phase_cycle |
| **Texts** | 11 | year_events, exercise_list, time_override, birthday, kids, parents, siblings, milestones, partner_ranges, marriage_ranges, pomo_test_phase |
| **Buttons** | 1 | pomodoro_start |
| **Text Sensors** | 2 | pomodoro_event, pomodoro_exercise |

All entities use NVS persistence — settings survive reboots without YAML `restore_value`.

## Icons

Animated icons (8×8) can be defined in YAML and rendered anywhere in the display:

**Sources:**
- `lameid:` — Download from LaMetric (e.g., https://developer.lametric.com/icons/32866)
- `file:` — Local GIF/PNG file
- `url:` — Remote URL

**Animation:** GIFs are fully supported with frame timing preserved. Icons animate continuously.

**Usage in C++:**
```cpp
draw_icon(it, "play", x, y);                    // Default color
draw_icon(it, "play", x, y, Color(255, 0, 0));  // Tinted
if (has_icon("play")) { ... }                   // Check existence
```

## Migration from v1.x

If upgrading from v1.x, remove all manual entity definitions from your YAML:

**Remove these blocks:**
- `switch:` with template switches (show_year, show_month, etc.)
- `select:` with template selects (style, gradient_type, etc.)
- `number:` with template numbers (brightness, cycle_time, etc.)
- `text:` with template texts (year_events, ls_birthday, etc.)
- `button:` with pomodoro_start
- `text_sensor:` with pomodoro_event, pomodoro_exercise
- OTA `on_begin`, `on_progress`, `on_end` handlers
- `on_boot` lambda that wired entities manually

**Add to life_matrix block:**
- `display: matrix_display` (for brightness control)
- `lifespan:` block with biographical data
- `year_events:` and `exercise_list:` for initial values

See [`example-full.yaml`](example-full.yaml) for a complete working configuration.
